= Part 2: Building GraphRAG Agents

This part adds the unstructured layer and builds agents that retrieve across both layers of the knowledge graph.

You will use the Strands Agents SDK with Amazon Bedrock to build agents that combine vector search and graph traversal for retrieval. The labs progress from a basic agent with simple tools, to graph-enriched vector retrieval using the neo4j-graphrag library, to connecting agents to the Neo4j knowledge graph through MCP servers.

image::two-layer-graph.png[The Two-Layer Graph,800]

== The neo4j-graphrag Library

Lab 4 uses the https://github.com/neo4j/neo4j-graphrag-python[neo4j-graphrag] Python library, which provides the building blocks for GraphRAG applications on Neo4j. The library includes retriever classes (`VectorRetriever`, `VectorCypherRetriever`) that handle embedding, vector search, and graph traversal, a `GraphRAG` pipeline class that combines retrieval with LLM generation, and pluggable LLM and embedder abstractions for providers including Bedrock, OpenAI, and Anthropic. Lab 6 uses the library's knowledge graph construction components to build the data pipeline from scratch.
